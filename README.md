# OrcOutFit

Нативный **ASI-плагин** для **GTA San Andreas 1.0 US** (x86), в том числе для **SA:MP**. Рисует оружие на теле педа, кастомные объекты и кастомные скины поверх модели, с настройкой через **ImGui**-оверлей и INI-файлы.

---

## Возможности

### Оружие на теле
- Отображение моделей оружия на выбранной **кости** (RpHAnim node ID), со смещением, поворотом и масштабом.
- Оружие в **активном слоте** (в руках) на теле **не дублируется**.
- Секции по имени оружия из игры и резервные секции `[WeaponNN]` для кастомных типов.

### Все педы в радиусе (опционально)
- Режим **«оружие у всех педов»** с ограничением по **радиусу** от локального игрока.
- Отдельный кэш инстансов на каждого педа.

### Кастомные объекты
- Сканирование **`*.dff`** в папке `OrcOutFit\object`.
- Для каждого объекта — свой **`<имя>.ini`**: кость, смещение, поворот, масштаб, вкл/выкл.

### Скины
- Сканирование **`*.dff`** в `OrcOutFit\SKINS` (корень) — список скинов в UI.
- Рендер выбранного **clump** поверх педа, привязка анимации через **RpSkin**.
- **Скрытие базового педа** (альфа clump) — опция.
- **SA:MP: привязка по нику** — в INI скина список ников; приоритет выше случайных пулов.
- **«Мой ник → выбранный скин»** — для локального игрока приоритет выбранного скина из списка (если включено), но **скин по нику всегда выше** этой опции и рандома.
- **Случайные варианты по модели**: папка `OrcOutFit\SKINS\random\<имя_модели_педа>\` с несколькими `.dff`/`.txd`; для **каждого** педа с этим `model id` один раз выбирается случайный вариант и закрепляется, пока пед жив в пуле. Имя подпапки должно совпадать с именем модели в игре (например `wmyclot`), проверка через `CModelInfo::GetModelInfo` и тип **ped**.

### Переопределение оружия по модели педа
- Папка `OrcOutFit\weaponsetting`: файлы **`<имя_модели>.ini`** (как в IDE), набор секций оружия как в основном INI.

### Интерфейс и SA:MP
- Окно **ImGui** с вкладками **Weapons / Objects / Skins**, сохранение в INI из UI.
- В **SA:MP** при **известной** сборке `samp.dll`: переключение курсора через **`SetCursorMode`** (как в типовых решениях), чат-команда на открытие меню (хук на отправку команды).
- Если версия **SA:MP не из списка** — плагин ведёт себя как в **одиночной игре** (клавиша из INI, без API ников/курсора SA:MP).

### Лог
- Рядом с ASI создаётся **`OrcOutFit.log`** (отладочные сообщения, ошибки загрузки и т.д.).

## Установка

1. Соберите проект (см. раздел [Сборка](#сборка)) или возьмите готовый **`OrcOutFit.asi`**.
2. Положите **`OrcOutFit.asi`** в каталог с игрой (или используйте modloader — пути данных считаются **от каталога ASI**).
3. Рядом с ASI появится **`OrcOutFit.ini`** (при первом запуске может создаться дефолтный конфиг).
4. Создайте при необходимости каталоги (относительно папки с ASI):
   - `OrcOutFit\object`
   - `OrcOutFit\SKINS`
   - `OrcOutFit\SKINS\random\<имя_модели>\` — для рандом-пулов
   - `OrcOutFit\weaponsetting`

---

## Сборка

- Visual Studio с компонентом **MSBuild**, платформа решения: **`x86`** (в `.vcxproj` это Win32).
- Пример (путь к MSBuild подстройте под свою установку VS):

```bat
"C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe" OrcOutFit.sln /p:Configuration=Release /p:Platform=x86
```

Путь к `MSBuild.exe` может отличаться (другая редакция или год VS) — возьмите актуальный из каталога установки Visual Studio.

Артефакт по умолчанию: **`build\Release\OrcOutFit.asi`**.

Зависимости в репозитории: **plugin-sdk**, **imgui**, **MinHook** (в `source\external\`).

## Горячие клавиши и чат

| Режим | Открытие меню |
|--------|----------------|
| Одиночная игра | Клавиша из **`OrcOutFit.ini`** → `[Main]` → **`ActivationKey`** (по умолчанию **F7**) |
| SA:MP | Команда из **`Command`** (по умолчанию **`/orcoutfit`**). Дополнительно можно включить **`SampAllowActivationKey=1`**, чтобы работала и клавиша **ActivationKey** |

В меню удержание **ПКМ** временно отдаёт управление камере игры (меню остаётся открытым).

## Файл `OrcOutFit.ini`

### Секция `[Main]`

| Ключ | Описание |
|------|----------|
| `Enabled` | Включён ли плагин (`0` / `1`) |
| `RenderAllPedsWeapons` | Рисовать оружие у педов в радиусе (`0` / `1`) |
| `RenderAllPedsRadius` | Радиус в метрах (целое, по умолчанию `80`) |
| `ActivationKey` | Имя клавиши для одиночки (например `F7`) |
| `SampAllowActivationKey` | В SA:MP разрешить и клавишу, и чат (`0` / `1`) |
| `Command` | Строка команды в чат SA:MP (должна начинаться с `/`) |

### Секции оружия

- По имени типа оружия из игры, например `[M4]`, либо **`[WeaponNN]`** для числового `eWeaponType`.
- Типичные поля: `Enabled`, `Bone`, `OffsetX/Y/Z`, `RotationX/Y/Z`, `Scale`.

### Секция `[SkinMode]`

| Ключ | Описание |
|------|----------|
| `Enabled` | Режим скинов (`0` / `1`) |
| `HideBasePed` | Скрывать исходный clump педа (`0` / `1`) |
| `NickMode` | Использовать ники SA:MP для привязки скинов (`0` / `1`) |
| `LocalPreferSelected` | Для локального игрока предпочитать выбранный в UI скин, если ник не дал свой скин (`0` / `1`) |
| `RandomFromPools` | Включить случайные варианты из `SKINS\random\...` (`0` / `1`) |
| `Selected` | Имя выбранного скина из корня `SKINS` (как имя `.dff` без расширения) |

---

## INI кастомного объекта (`OrcOutFit\object\<имя>.ini`)

Секция **`[Main]`**: `Enabled`, `Bone`, `OffsetX/Y/Z`, `RotationX/Y/Z`, `Scale` — аналогично оружию.

---

## INI кастомного скина (`OrcOutFit\SKINS\<имя>.ini` или в подпапке `random\...`)

Секция **`[NickBinding]`**:

| Ключ | Описание |
|------|----------|
| `Enabled` | Включить привязку по нику (`0` / `1`) |
| `Nicks` | Ник(и): через запятую и/или **с новой строки**; сравнение без учёта регистра |

---

## INI в `OrcOutFit\weaponsetting\<модель>.ini`

Структура как у основного `OrcOutFit.ini`: те же секции оружия; применяется, когда у педа модель с ключом, совпадающим с именем файла (например `lapd1.ini`).

---

## Поддержка SA:MP

- Определяется **entry point** `samp.dll`; для известных клиентов доступны: **хук команды**, **ник по педу**, **`SetCursorMode`** для курсора в меню.
- Неизвестная сборка: **никаких** вызовов по оффсетам SA:MP — только поведение как в SP (клавиша, патч курсора GTA при необходимости).

## Ограничения и заметки

- Жёстко ориентировано на **GTA SA 1.0 US** (адреса патчей курсора и т.д.).
- Качество скина зависит от совместимости **скелета/иерархии** с моделью педа; в лог пишутся проблемы загрузки и иерархии.
- Не копируйте ASI в игру автоматически из скриптов сборки, пока запущена игра — файл может быть занят процессом.
